STM32简介

ST即“ST公司”，M即microcontroller（mcu，单片机）的首字母，32即32位

新建工程

开发方式

  • 基于寄存器

  • 基于标准库

  • 基于HAL库的方式

GPIO(General Purpose Input Output)通用输入输出口

  • 可配置为8种输入输出模式

  • 引脚电平：0V~3.3V，部分引脚可容忍5V（参考引脚定义，带ft是可以容忍5V的）

  • 输出模式下可控制端口输出高低电平，用以驱动LED、控制蜂鸣器、模拟通信协议输出时序等

  • 输入模式下可读取端口的高低电平或电压，用于读取按键输入、外接模块电平信号输入、ADC电压采集、模拟通信协议接收数据等

基本结构

内存可以通过APB2总线对寄存器进行读写，这样就可以完成对GPIO端口的读写，寄存器是32位，但是端口只有16个，对应低16位字段

涉及两个外设RCC和GPIO，可以在Library-rcc.h后面查看RCC库函数的定义，常用的有(右键跳转到定义可以查看函数说明和参数信息)

void RCC_AHBPeriphClockCmd(uint32_t RCC_AHBPeriph, FunctionalState NewState);

void RCC_APB2PeriphClockCmd(uint32_t RCC_APB2Periph, FunctionalState NewState);

void RCC_APB1PeriphClockCmd(uint32_t RCC_APB1Periph, FunctionalState NewState);

gpio.h库函数常用的有：

void GPIO_SetBits(GPIO_TypeDef* GPIOx, uint16_t GPIO_Pin);:将指定端口设置为高电平

void GPIO_ResetBits(GPIO_TypeDef* GPIOx, uint16_t GPIO_Pin);：将指定端口设置为低电平

void GPIO_WriteBit(GPIO_TypeDef* GPIOx, uint16_t GPIO_Pin, BitAction BitVal);：将指定端口设置为指定电平，即包含前两种函数功能

void GPIO_Write(GPIO_TypeDef* GPIOx, uint16_t PortVal);：直接把值写进对应端口的寄存器，下面流水灯程序会用到

在推挽模式下二极管怎么接都可闪烁，说明推挽模式高低电平都具有驱动能力。如果设置为开漏输出，则发现高电平点亮时LED不量说明开漏输出的高电平没有驱动能力。

LED流水灯

可以用或操作初始化多个端口，根据定义，换算成二进制每个端口对应一位，例如 PIN_0 | PIN_1 就是

0000 0000 0000 0001 | 0000 0000 0000 0010 即 0000 0000 0000 0011

对应位为1即可选中该端口，很多参数都可以这样操作

中断系统

  • 中断系统

主程序运行过程中，出现了特定的中断触发条件（中断源），使得CPU暂停转而去处理中断（运行一个中断处理程序），中断处理完成后回到原程序暂停的位置继续执行

  • 中断优先级

多个中断源同时发出中断请求后，CPU会根据中断源的轻重缓急进行裁决，优先响应更加紧急的中断源

  • 中断嵌套

当一个中断程序正在运行时，又有新的更高优先级的中断源申请中断，CPU再次暂停当前中断程序，转而去处理新的中断程序，处理完成后依次进行返回

  • 中断执行流程

在发生中断时需要保存当前程序上下文，以便之后回来接着执行

STM32中断

  • 68个可屏蔽中断通道，包含EXTI、TIM、ADC、USART、SPI、I2C、RTC等多个外设

  • 使用NVIC统一管理中断（帮助CPU判断该处理哪个中断），每个中断通道都拥有16个可编程的优先等级，可对优先级进行分组，进一步设置抢占优先级和响应优先级

优先级分组

  • NVIC的中断优先级由优先级寄存器的4位（0~15）决定，这4位可以进行切分，分为高n位的抢占优先级和低4-n位的响应优先级

  • 抢占优先级高的可以中断嵌套，响应优先级高的可以优先排队，抢占优先级和响应优先级均相同的按中断号排队

EXTI（Extern Interrupt）外部中断

  • EXTI可以监测指定GPIO口的电平信号，当其指定的GPIO口产生电平变化时，EXTI将立即向NVIC发出中断申请，经过NVIC裁决后即可中断CPU主程序，使CPU执行EXTI对应的中断程序

  • 支持的触发方式：上升沿/下降沿/双边沿/软件触发

  • 支持的GPIO口：所有GPIO口，但相同的Pin（例如PA0和PB0,PA1和PB1等等）不能同时触发中断

  • 通道数：16个GPIO_Pin，外加PVD输出、RTC闹钟、USB唤醒、以太网唤醒

  • 触发响应方式：中断响应/事件响应

AFIO复用IO口

  • AFIO主要用于引脚复用功能的选择和重定义（在开始引脚定义表，如果把默认复用功能换成重定义栏的功能，就是用AFIO来完成）

  • 在STM32中，AFIO主要完成两个任务：复用功能引脚重映射、中断引脚选择

对射式红外传感器计数

  • 在Hardware里添加CountSensor.c和.h文件

初始化函数参考EXTI基本结构图，需要将所有参与的外设都配置，依次有GPIO、AFIO、EXTI、NVIC（EXIT和NVIC时钟一直开启，不需要手动打开，NVIC是内核外设）

AFIO相关库函数也在GPIO.c、.h中，在GPIO.h中下拉可以找到

void GPIO_PinRemapConfig(uint32_t GPIO_Remap, FunctionalState NewState); void GPIO_EXTILineConfig(uint8_t GPIO_PortSource, uint8_t GPIO_PinSource);

GPIO_PinRemapConfig用于引脚重映射，GPIO_EXTILineConfig就是中断函数，可以配置AFIO的数据选择器从而选择我们的中断引脚

TIM输出比较OC（Output Compare）

  • 输出比较可以通过比较CNT与CCR寄存器值的关系，来对输出电平进行置1、置0或翻转的操作，用于输出一定频率和占空比的PWM波形

  • 每个高级定时器和通用定时器都拥有4个输出比较通道

  • 高级定时器的前3个通道额外拥有死区生成和互补输出的功能

PWM简介

  • PWM（Pulse Width Modulation）脉冲宽度调制

  • 在具有惯性的系统中，可以通过对一系列脉冲的宽度进行调制，来等效地获得所需要的模拟参量，常应用于电机控速等领域

  • PWM参数：

频率 = 1 / Ts 越高越稳定，但是性能开销越大
占空比 = TON / Ts
分辨率 = 占空比变化步距
