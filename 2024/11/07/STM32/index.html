STM32

发表于 2024-11-07

STM32简介

ST即“ST公司”，M即microcontroller（mcu，单片机）的首字母，32即32位

新建工程

开发方式

  • 基于寄存器

  • 基于标准库

  • 基于HAL库的方式

GPIO(General Purpose Input Output)通用输入输出口

  • 可配置为8种输入输出模式

  • 引脚电平：0V~3.3V，部分引脚可容忍5V（参考引脚定义，带ft是可以容忍5V的）

  • 输出模式下可控制端口输出高低电平，用以驱动LED、控制蜂鸣器、模拟通信协议输出时序等

  • 输入模式下可读取端口的高低电平或电压，用于读取按键输入、外接模块电平信号输入、ADC电压采集、模拟通信协议接收数据等

基本结构

内存可以通过APB2总线对寄存器进行读写，这样就可以完成对GPIO端口的读写，寄存器是32位，但是端口只有16个，对应低16位字段

GPIO模式

LED闪烁

1.使用RCC开启GPIO时钟

2.使用GPIO_Init初始化GPIO

3.使用输入输出函数控制I/O

涉及两个外设RCC和GPIO，可以在Library-rcc.h后面查看RCC库函数的定义，常用的有(右键跳转到定义可以查看函数说明和参数信息)

void RCC_AHBPeriphClockCmd(uint32_t RCC_AHBPeriph, FunctionalState NewState);

void RCC_APB2PeriphClockCmd(uint32_t RCC_APB2Periph, FunctionalState NewState);

void RCC_APB1PeriphClockCmd(uint32_t RCC_APB1Periph, FunctionalState NewState);

gpio.h库函数常用的有：

void GPIO_SetBits(GPIO_TypeDef* GPIOx, uint16_t GPIO_Pin);:将指定端口设置为高电平

void GPIO_ResetBits(GPIO_TypeDef* GPIOx, uint16_t GPIO_Pin);：将指定端口设置为低电平

void GPIO_WriteBit(GPIO_TypeDef* GPIOx, uint16_t GPIO_Pin, BitAction BitVal);：将指定端口设置为指定电平，即包含前两种函数功能

void GPIO_Write(GPIO_TypeDef* GPIOx, uint16_t PortVal);：直接把值写进对应端口的寄存器，下面流水灯程序会用到

在推挽模式下二极管怎么接都可闪烁，说明推挽模式高低电平都具有驱动能力。如果设置为开漏输出，则发现高电平点亮时LED不量说明开漏输出的高电平没有驱动能力。

LED流水灯

可以用或操作初始化多个端口，根据定义，换算成二进制每个端口对应一位，例如 PIN_0 | PIN_1 就是

0000 0000 0000 0001 | 0000 0000 0000 0010 即 0000 0000 0000 0011

对应位为1即可选中该端口，很多参数都可以这样操作

GPIO输入-按键控制

按键在按下时会有一段抖动，需要进行过滤，不然LED会在短时间连续闪烁几次，最简单的方法是加个延时
