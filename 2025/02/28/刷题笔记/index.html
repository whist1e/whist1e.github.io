贪心算法

贪心算法：每一步都做出在当前情况下最好的选择，也就是说，不从整体最优上加以考虑，算法得到的是在某种意义上的局部最优解。
贪心算法没有固定的算法框架，算法设计的关键是贪心策略的选择。必须注意的是，贪心算法不是对所有问题都能得到整体最优解，关键是贪心策略的选择。

1005.K次取反后最大化的数组和

给定一个整数数组 A，我们只能用以下方法修改该数组：我们选择某个索引 i 并将 A[i] 替换为 -A[i]，然后总共重复这个过程 K 次。（我们可以多次选择同一个索引 i。）

以这种方式修改数组后，返回数组可能的最大和。

思路很好想，每次都反转小的数字就好了，那么肯定是需要排序的。但是如果按值从小到大排序，会有很多问题：首先反转的肯定是最小的负数，那么当负数反转完了，K还大于0，这时候应该反转哪里的元素？应该是绝对值最小的元素

  • 第一步：将数组按照绝对值大小从大到小排序，注意要按照绝对值的大小
  • 第二步：从前向后遍历，遇到负数将其变为正数，同时K–
  • 第三步：如果K还大于0，那么反复转变数值最小的元素，将K用完
  • 第四步：求和

135. 分发糖果

n 个孩子站成一排。给你一个整数数组 ratings 表示每个孩子的评分。

  • 每个孩子至少分配到 1 个糖果。
  • 相邻两个孩子评分更高的孩子会获得更多的糖果。

请你给每个孩子分发糖果，计算并返回需要准备的最少糖果数目。

这道题目一定是要确定一边之后，再确定另一边，例如比较每一个孩子的左边，然后再比较右边，如果两边一起考虑一定会顾此失彼。
先确定右边评分大于左边的情况(也就是从前向后遍历，填充candy数组)

局部最优：如果右边评分比左边大，右边的孩子就多一个糖果

全局最优：相邻的孩子中，右边的孩子比左边的孩子糖果更多（一半的逻辑已经完成了）

然后再从后往前遍历，完成如果左边评分比右边大，左边的孩子就多一个糖果

406.根据身高重建队列

假设有打乱顺序的一群人站成一个队列，数组 people 表示队列中一些人的属性（不一定按顺序）。每个 people[i] = [hi, ki] 表示第 i 个人的身高为 hi ，前面 正好 有 ki 个身高大于或等于 hi 的人。

请你重新构造并返回输入数组 people 所表示的队列。返回的队列应该格式化为数组 queue ，其中 queue[j] = [hj, kj] 是队列中第 j 个人的属性（queue[0] 是排在队列前面的人）。

主要学习思路，与分发糖果类似，在上题中，先确定右边的孩子比左边评分高就会多一个糖果，再实现左边的孩子比右边评分高则多一个糖果。

本题有两个维度，身高和前面比自己高的人，应该先完成一个逻辑，再实现另一个。

应该先将身高从大到小排序（身高相同，则ki小的在前面），方便第二个维度的确定。在完成排序后，再
从前往后遍历，因为此时前面的全比自己高，所以直接根据ki值将该值插入对应索引的位置，该点的相对位置就是正确的。即使后面的数插到了前面，因为后面的数更小，所以不影响该点的大小逻辑。

本题可以用链表优化性能，插入操作会比较麻烦；或者给vector定容，并手动给出insert的逻辑。

vector底层实现是数组，所以上述写法表面上时间复杂度是O(n)，但是真正复杂度是O(n^2)。vector插入新元素需要后面元素依次向后移动，并且vector动态数组的原理是，当前数组容量不够时，就会重新申请一个两倍容量的数组，再将原数组的值复制过来。当然对外暴露的长度只有capacity。

435. 无重叠区间

给定一个区间的集合 intervals ，其中 intervals[i] = [starti, endi] 。返回 需要移除区间的最小数量，使剩余区间互不重叠 。

注意 只在一点上接触的区间是 不重叠的。例如 [1, 2] 和 [2, 3] 是不重叠的。

与上一题射气球逻辑基本一样，选择对左边界进行排序，然后遍历得到有几个不重合的区间，然后总数减去最多的不重合区间，就是需要移除的数目。

我们记录当前重复区间的最小右边界，如果下一个区间左边界大于等于这个最小值，则说明不重复的区间又多了一个，因为记录的是最小值，可以保证最后得到的不重复区间是最多的，也就是移除的区间是最少的。

968.监控二叉树

首先，摄像头一定没有放在叶子节点上，所以从下往上看，局部最优是让每个节点的父节点上安装上摄像头，整体最优是用最少的摄像头覆盖整个二叉树。遍历顺序是后序遍历

关于如何确定一个要不要放置摄像头，本题中的节点应该有三种状态

  • 0：无摄像头覆盖
  • 1：有摄像头覆盖
  • 2：该节点上有摄像头
    关于空节点的状态，如果空节点是无覆盖，则叶子节点就需要放置摄像头了，不符合上面的分析，所以空节点是有覆盖的状态

然后是通过左右孩子的状态来判断父节点对应的操作

  1. 左右孩子有摄像头覆盖（左右孩子的孩子上有摄像头），则该节点是无覆盖的状态（0），应该在该节点的父节点放置摄像头，该节点不需要放置摄像头

  2. 左右孩子至少有一个未被覆盖，则该节点需要放置摄像头

  3. 左右孩子至少有一个有摄像头，则该节点是有覆盖的状态（1），注意此时有一种情况：左右孩子有一个是无覆盖状态，另一个是有摄像头，这种情况应该是按情况2处理还是情况3处理？应该是2，所以情况2的判断要在前面！

  4. 头节点需要单独进行判断，因为如果头节点的左右孩子都被覆盖，因为头节点没有父节点，则按照上面的原则头节点是无法被监控到的，需要放一个摄像头

关于循环如何写，首先是

  • 参数和返回值，应该传入的是一个节点，返回值是该节点的状态
  • 终止条件是遇到空节点，应该返回1（有覆盖），这样就可以在叶子节点的父节点放置摄像头

爬到第一层有1种方法，爬到第二层有两种方法，那么第一层跨越两步就到了第三层，第二层跨越一步就到了第三层，如此当前楼梯的方法数可以由前面两种状态推导得到。

  1. dp[i]表示第i层楼梯有几种方法爬上
  2. dp[i] = dp[i - 1] + dp[i - 2]
  3. dp[1] = 1，dp[2] = 2，至于dp[0]不做讨论，直接从3开始遍历。
  4. 从后往前遍历

746. 使用最小花费爬楼梯

需要一个数组来存储花费的精力cost

  1. dp[i]表示爬上当前楼梯的总cost
  2. dp[i] = dp[i - 1] + min(cost(n + 1), cost(n + 2))
  3. dp[0] = min(cost[0], cost[1]), 从小的下标开始遍历
  4. 从前往后遍历
  5. dp[0] = 0, dp[1] = dp[0] + min(cost(0), cost(1)), dp[2] = min(cost())

错误思考，无法正确的遍历。。正确的是dp[i]是爬上当前楼梯所花费的最小精力，那么dp可以选择从dp[i - 1]或者dp[i - 2]爬上来，具体选择哪个取决于dp[i - 1] + cost[i - 1] 与 dp[i - 2] + cost[i - 2]，最后的结果也是从最后两阶台阶中进行选择

62.不同路径

动态规划首先要记录每一个格子的状态，dp[i]，表示到达该格子有多少种方法，每次只能向右或向下走一格，所以到达某个格子的方法可以由其上方和左侧的格式的方法相加得到

  1. dp[i][j]代表到达该坐标可以有多少种方法
  2. dp[i][j] = dp[i - 1][j] + dp[i][j - 1]
  3. dp[0][0] = 1
  4. 从下标（0，0）开始遍历二维数组

343. 整数拆分

  1. dp[i]表示拆分i可得到的最大乘数
  2. i可以拆分成很多数，
    • 如果是拆分成两个数就是 j * i - j
    • 拆分成多个数就是 j * dp[i - j]，我们只需要遍历确定 j ，就可得到最大的乘数, 并且不必拆分成 dp[i] * dp[i - j]，因为遍历确定 j 时已经包含了所有情况
  3. dp[0], dp[1], 本身没有意义，并且拆分成0，1的情况乘积一定不会是最大值，所以只初始化dp[2] = 1;
  4. 从3开始遍历，一直遍历到n
  5. 刚开始以为j * dp[i - j]包含了所有情况，忽略了 j * (i - j) 可能更大，在错误后打印结果结果发现在 4 这里出了问题，4 分解 2 * 2 是最大的，而 2 * dp[2] = 3，导致后续结果全是错的

96.不同的二叉搜索树

  1. dp[i]表示i个节点组成的二叉搜索树有多少种，最终的输出结果就是dp[n]

  2. 以 n = 3 为例，有以下几种情况

    • 3为头结点时，左侧有两个结点，并且分布于dp[2]一致（数值不一样没关系，要的只是有多少种情况）
    • 2为头节点时，左右各一个节点，都与dp[1]分布一致
    • 1为头节点时，右侧有两个结点，并且分布于dp[2]一致

    所以dp[3] = dp[2] * dp[0] + dp[1] * dp[1] + dp[0] * dp[2]，一定是乘法，比如左子树的一种情况对应右子树的n种情况，就有n个结果，这就是状态转移方程。当 j 为头节点时，左子树有 j - 1 个数，就有dp[j - 1] 种情况，右子树有 i - j 个数，有dp[i - j]种情况（虽然数字不同，但是个数相同的 n 个数组成的二叉搜索树数目相同）。

  3. dp[0] = 1

  4. 从1开始遍历到n

01背包

有n件物品和能空间为m的背包，第i件物品的重量为weight[i]，价值为value[i]，求背包所能装下的物品的最大价值。每件物品只有一个。

暴力解法是不断回溯，每件物品都有两种状态–装或不装，回溯每件物品的状态时间复杂度是2^n

动态规划可以使用一维数组或二维数组，先看二维状态数组

  1. dp[i][j]，表示在 0-i 的物品内任选，空间为 j 所装下的最大价值

  2. dp[i][j]的推导：

    • 当不装 i 这件物品时，背包内最大价值为dp[i - 1][j]
    • 装 i 这件物品时，最大价值为 dp[i - 1][j - weight[i]] + value[i]，解释是装下 i 后，背包剩余空间为 j - weight[i] ，剩余空间所能装的最大价值就是 dp[i - 1][j - weight[i]]，

    dp[i][j] 就是这两种情况取最大值

  3. 由上面的推导可以看出， dp[i][j] 可以由它上方和左上方的dp推导而来，至于初始化，可根据 dp 的意义，j = 0 时表示背包容量为 0 ，能装的价值自然是 0 ，即 j = 0 这一列都为0；i = 0 表示只装物品 0 ，但是背包容量小于 weight[0] 时装不下，所以 dp[0][j] = 0 ，当j > weight[0], dp[0][j] = value[0]；

  4. 对于本二维状态数组的遍历，背包和物品谁先后都可以，因为 dp[i][j] 可以由它上方和左上方的dp推导而来，无论从哪个方向遍历都满足要求

再看一维数组，与二维数组类似，但是需要考虑更多细节

  1. dp[j] 表示容量为j的背包当前可背的最大价值是dp[j]，此数组在i递增时会不断滚动刷新，所达到的效果与二维一样。只是将矩阵压缩成了一行
  2. dp[j] 继承自 dp[j - 1] ，不装当前物品时，背包内最大价值就是 dp[j]；装入该物品时，最大价值为dp[j - weight[i]] + value[i]
  3. 数组初始化为0即可，并且 dp 数组长度是 n+1 ，因为需要用到背包容量为0时的值。二维数组因为要用到dp[i - 1] 的值，所以必须初始化第0件物品，然后从第1件物品开始遍历
  4. 遍历顺序一定是先遍历物品，再遍历背包；并且遍历背包时需要倒序遍历，这是由于后面的 j 在计算dp[j] 时需要计算dp[j - weight[i]] + value[i]，如果从前往后遍历前面的数组已经更新为可以装入i是的最大容量，这时后面的 j 再计算装入时的最大价值dp[j - weight[i]] + value[i]就不准了，因为前面的也装入了物品 i ；

水流问题

现有一个 N × M 的矩阵，每个单元格包含一个数值，这个数值代表该位置的相对高度。矩阵的左边界和上边界被认为是第一组边界，而矩阵的右边界和下边界被视为第二组边界。

矩阵模拟了一个地形，当雨水落在上面时，水会根据地形的倾斜向低处流动，但只能从较高或等高的地点流向较低或等高并且相邻（上下左右方向）的地点。我们的目标是确定那些单元格，从这些单元格出发的水可以达到第一组边界和第二组边界。

一般想法可以是遍历每一个点，dfs bfs均可，看这些点是否可以到达两个边界。

遍历每一个节点，是 m * n，遍历每一个节点的时候，都要做深搜，深搜的时间复杂度是： m * n，整体时间复杂度 就是 O(m^2 * n^2)；

优化思路是先从第一组边界出发，逆流而上看它可以到达那些节点并标记。再从第二组边界出发，逆流而上看它可以到达那些节点并标记，两组边界都标记过的节点就是结果，即可以到达两组边界。

因为被标记过的点不会再访问，所以只会遍历两次所有点（2 * m * n），第一组边界遍历一次，第二组边界遍历一次

并查集

当我们需要判断两个元素是否在同一个集合里的时候，我们就要想到用并查集。

  • 将两个元素添加到一个集合中。
  • 判断两个元素在不在同一个集合

如何表示一个集合？简单的就是同属一个集合放到一个vector、map、set，但是当有很多集合时这样需要定义很多结构；假设 ABC 是一个集合，可以定义一个数组father，father[A] = B，father[B] = C，father[C] = C，这样就可以表示 ABC 是一个集合，因为它们的根相同，都是B。

如何判断两个元素是否在同一个集合？只需要判断它们的根是否相同即可。例如find(A) = find(B) = C

其实上述结构就是树，那么当树很深的时候，找根就需要不断递归，但其实，中建变量我们都可以省略，比如上述就可以压缩为 father[A] = C，father[B] = C，father[C] = C，这样只需要一次递归就可以找到根。

一个误区：

上述 isSame 和 join 看着有重复的代码，是否可以简写为

其实是不对的，假如执行 join(1, 2) 和 join(3, 2)，按照上述写法，father[2] = 1，father[2] = 3，这时isSame(1, 3)，find(1) 返回的是1，find(3)返回的是3。 return 1 == 3 返回的是false，代码告诉我们 1 和 3 不在同一个集合，这肯定是不对的。

问题出在每次join的时候都要先找到根，正确过程是 join(1, 2)后 father[2] = 1，join(3, 2) 时先找到 2 的根是1，再执行 father[1] = 3，这样 isSame(1, 3) find(1) 返回的是1 find(3)返回的是也是 1。

prim算法（最小生成树）

有 n 个节点，使用 n - 1 个节点一定可以将它们连通，但是要保证权值最小。

prim 算法是节点的贪心角度出发，每次寻找离当前最小生成树最近（权值最小）的节点加入：

  • 第一步，选距离生成树最近节点
  • 第二步，最近节点加入生成树
  • 第三步，更新非生成树节点到生成树的距离（即更新minDist数组）
    所以计算和更新 minDist 就是关键

首先选取一个点作为最小生成树的起点，然后更新 minDist 中与该点相连的点的最小距离，然后选权值最小的点加入最小生成树，再更新 minDist 中与该点相连的点的最小距离…

直到所有点都加入后， minDist 中保存的就是该树各边权值

拓扑排序

给出一个 有向图，把这个有向图转成线性的排序 就叫拓扑排序。

当然拓扑排序也要检测这个有向图 是否有环，即存在循环依赖的情况，因为这种情况是不能做线性排序的。

0 不依赖任何节点，它的入度是 0，出度是 2。拓扑排序过程就是这样：

  • 找到入度为0 的节点，加入结果集
  • 将该节点从图中移除

还剩下一个问题就是如何判断循环依赖（环），当没有入度为0的节点，并且图中还有节点（或结果集里没有收集到所有节点）则是有环。
