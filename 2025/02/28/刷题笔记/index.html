组合总和 II

难点在于如何去重，数组中可能有重复的元素，但是结果是组合，不能重复。并且结果中可以出现相同的值，因为它们是足足中不同的元素，只是值相同。例如(1,1,2)
解决办法：

  • 先排序，使相同元素相邻，然后遍历时，如果当前元素和前一个元素相同，则跳过，防止重复。（相同元素再遍历，如果有结果一定与前面的重复）
  • 使用一个used数组来标记当前元素是否被使用过，每次需要判断used[i - 1]是否被使用过，如果谓未被使用过（说明从第二个1，开始了新的遍历），则跳过，防止重复。被使用过则说明本次递归是从之前的元素开始的，可以继续递归。

贪心算法

贪心算法：每一步都做出在当前情况下最好的选择，也就是说，不从整体最优上加以考虑，算法得到的是在某种意义上的局部最优解。
贪心算法没有固定的算法框架，算法设计的关键是贪心策略的选择。必须注意的是，贪心算法不是对所有问题都能得到整体最优解，关键是贪心策略的选择。

122.买卖股票的最佳时机 II

  • 只有一只股票！
  • 当前只有买股票或者卖股票的操作
  • 想获得利润至少要两天为一个交易单元。
    这道题有点像区间和题目，使用前缀和数组，当然逻辑不一样。最终利润是可以分解的，
    假如第 0 天买入，第 3 天卖出，那么利润为：prices[3] - prices[0]。

相当于(prices[3] - prices[2]) + (prices[2] - prices[1]) + (prices[1] - prices[0])。拆分成了第一二三天分别的利润相加（第0天没有利润）

第一天当然没有利润，至少要第二天才会有利润，所以利润的序列比股票序列少一天！

从图中可以发现，其实我们需要收集每天的正利润就可以，收集正利润的区间，就是股票买卖的区间，而我们只需要关注最终利润，不需要记录区间。

55. 跳跃游戏

给定一个非负整数数组，你最初位于数组的第一个位置。

数组中的每个元素代表你在该位置可以跳跃的最大长度。

判断你是否能够到达最后一个位置。

每次跳几步不重要，关键是争取最大覆盖范围。能覆盖到最后一个元素为true，否则为false。(注意长度为1的数组本身就在终点了)，只需要一个cover变量辅助，cover表示当前可覆盖到的范围，所以循环判断条件也是，当cover>=nums.size(),return tru;当遍历到cover外还没返回，表示到不了终点。

45.跳跃游戏 II

给定一个非负整数数组，你最初位于数组的第一个位置。

数组中的每个元素代表你在该位置可以跳跃的最大长度。

你的目标是使用最少的跳跃次数到达数组的最后一个位置。

这里需要统计两个覆盖范围，当前这一步的最大覆盖和下一步最大覆盖。当当前覆盖范围未到终点时，再从当前覆盖范围中选出下一步的覆盖范围。

1005.K次取反后最大化的数组和

给定一个整数数组 A，我们只能用以下方法修改该数组：我们选择某个索引 i 并将 A[i] 替换为 -A[i]，然后总共重复这个过程 K 次。（我们可以多次选择同一个索引 i。）

以这种方式修改数组后，返回数组可能的最大和。

思路很好想，每次都反转小的数字就好了，那么肯定是需要排序的。但是如果按值从小到大排序，会有很多问题：首先反转的肯定是最小的负数，那么当负数反转完了，K还大于0，这时候应该反转哪里的元素？应该是绝对值最小的元素

  • 第一步：将数组按照绝对值大小从大到小排序，注意要按照绝对值的大小
  • 第二步：从前向后遍历，遇到负数将其变为正数，同时K–
  • 第三步：如果K还大于0，那么反复转变数值最小的元素，将K用完
  • 第四步：求和

134. 加油站

在一条环路上有 N 个加油站，其中第 i 个加油站有汽油 gas[i] 升。

你有一辆油箱容量无限的的汽车，从第 i 个加油站开往第 i+1 个加油站需要消耗汽油 cost[i] 升。你从其中的一个加油站出发，开始时油箱为空。

如果你可以绕环路行驶一周，则返回出发时加油站的编号，否则返回 -1。

记录加油站消耗与补充的差值，并且从前面开始累加，如果累加到某个加油站剩余油量小于0，说明这个站点及前面的站点都不能作为起点，我们就从这个站点之后的站点作为起点。如果从该点开始往后计算剩余油量没有出现负数，则该点就是结果（除了没有结果的情况）。如果出现了负数，则继续上述逻辑（贪心的逻辑）。

135. 分发糖果

n 个孩子站成一排。给你一个整数数组 ratings 表示每个孩子的评分。

  • 每个孩子至少分配到 1 个糖果。
  • 相邻两个孩子评分更高的孩子会获得更多的糖果。

请你给每个孩子分发糖果，计算并返回需要准备的最少糖果数目。

这道题目一定是要确定一边之后，再确定另一边，例如比较每一个孩子的左边，然后再比较右边，如果两边一起考虑一定会顾此失彼。
先确定右边评分大于左边的情况(也就是从前向后遍历，填充candy数组)

局部最优：如果右边评分比左边大，右边的孩子就多一个糖果

全局最优：相邻的孩子中，右边的孩子比左边的孩子糖果更多（一半的逻辑已经完成了）

然后再从后往前遍历，完成如果左边评分比右边大，左边的孩子就多一个糖果

406.根据身高重建队列

假设有打乱顺序的一群人站成一个队列，数组 people 表示队列中一些人的属性（不一定按顺序）。每个 people[i] = [hi, ki] 表示第 i 个人的身高为 hi ，前面 正好 有 ki 个身高大于或等于 hi 的人。

请你重新构造并返回输入数组 people 所表示的队列。返回的队列应该格式化为数组 queue ，其中 queue[j] = [hj, kj] 是队列中第 j 个人的属性（queue[0] 是排在队列前面的人）。

主要学习思路，与分发糖果类似，在上题中，先确定右边的孩子比左边评分高就会多一个糖果，再实现左边的孩子比右边评分高则多一个糖果。

本题有两个维度，身高和前面比自己高的人，应该先完成一个逻辑，再实现另一个。

应该先将身高从大到小排序（身高相同，则ki小的在前面），方便第二个维度的确定。在完成排序后，再
从前往后遍历，因为此时前面的全比自己高，所以直接根据ki值将该值插入对应索引的位置，该点的相对位置就是正确的。即使后面的数插到了前面，因为后面的数更小，所以不影响该点的大小逻辑。

本题可以用链表优化性能，插入操作会比较麻烦；或者给vector定容，并手动给出insert的逻辑。

vector底层实现是数组，所以上述写法表面上时间复杂度是O(n)，但是真正复杂度是O(n^2)。vector插入新元素需要后面元素依次向后移动，并且vector动态数组的原理是，当前数组容量不够时，就会重新申请一个两倍容量的数组，再将原数组的值复制过来。当然对外暴露的长度只有capacity。

452. 用最少数量的箭引爆气球

本题的贪心局部最优是，尽可能让一只箭射爆更多气球。做法是先按左边界值进行排序（右边界也可以），目的是使可以一支箭射爆的气球一定是相邻的，方便遍历。

如果当前气球,的左边界小于上一次记录的最小边界，则说明可以与上次的气球一箭射爆

435. 无重叠区间

给定一个区间的集合 intervals ，其中 intervals[i] = [starti, endi] 。返回 需要移除区间的最小数量，使剩余区间互不重叠 。

注意 只在一点上接触的区间是 不重叠的。例如 [1, 2] 和 [2, 3] 是不重叠的。

与上一题射气球逻辑基本一样，选择对左边界进行排序，然后遍历得到有几个不重合的区间，然后总数减去最多的不重合区间，就是需要移除的数目。

我们记录当前重复区间的最小右边界，如果下一个区间左边界大于等于这个最小值，则说明不重复的区间又多了一个，因为记录的是最小值，可以保证最后得到的不重复区间是最多的，也就是移除的区间是最少的。

763. 划分字母区间

只涉及小写英文字母，先使用一个哈希数组来记录每次字母出现的最后位置
for (int i = 0; i < S.size(); i++) { //统计每一个字符最后出现的位置

56. 合并区间

如果有重叠区间将他们合并，看起来与上面的题差不多，但是实现起来难写很多。

首先依旧是按左边界进行排序，然后maxboard是当前区间的最大右边界，如果下一个区间左边界小于maxboard也需要被合并进来，反之是一个新的区间。

在合并的逻辑的逻辑上，我想的是每次找到新的区间之后再把整个区间放进result,最后写成了下面的逻辑，复杂且在一些边界处理上容易有坑

更好的思路是不需要start，先把第一个区间放进结果集，当有新区间需要合并，就更新result右区间。如下简洁了很多
